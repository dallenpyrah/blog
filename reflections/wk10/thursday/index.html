# Week 10 Day 4

In a SQL table, what is the difference between information in a row and information in a column?

Information in a row is the whole instance of that data. For example in our gregslist api we have different values on a car but when added to that table the whole row represents one car while the columns represent the many different things that belong to that car. Sql tables are pretty straight foward when it comes to viewing data in the table.

Demonstrate the basic structure for creating a new table called characters with the values name, age, description as strings, and an int id.

CREATE TABLE characters ( name VARCHAR(255) NOT NULL, age INT NOT NULL, description VARCHAR(255), id INT AUTO_INCREMENT

PRIMARY KEY (id);

)

What is the difference between the following statements:

DELETE FROM table_name;

This specific statement would delete all rows within our table but it would still allow us to keep the table, all of the data would be gone though.

DROP TABLE table_name;

This SQL statement would drop our whole table along with all the data in it. If we no longer want our table we can run this statement and the whole table would be gone.
